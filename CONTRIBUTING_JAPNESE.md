KASHIKA グローバル展開：分散型コントリビューションガイド
KASHIKAへようこそ。本プロジェクトは中央集権的なプラットフォームではありません。歴史的・政治的ファクトをフラットに結合する、「自律分散型の並列マトリクス（格子）」です。

ここでは、国同士の上下関係やランク付け、主観的な解釈は一切行いません。私たちが繋ぐのは「感情を排した1次ソース付きのデータ構造」のみです。あなたの国のゲート（リポジトリ）を開設することで、その国が持つ過去の「前例（実績ログ）」が、他国の未来の政策を自動的にバックアップ（または矛盾を検知）する世界共通のインフラとなります。

🌍 独立ゲートの開設方法（3分でできるスターターキット）
KASHIKAのグローバル・レジストリに参加するには、このリポジトリをフォークするか、kashika-gate-[国際電話国番号] という名前の独立したパブリックリポジトリ（例：台湾なら kashika-gate-886）を新規作成してください。

自走を開始するために必要なファイルは、実質以下の2つのJSONだけです。

① /index.json（ゲート目録）
リポジトリのルート（直下）に配置します。これにより、世界共通規格である kashika-core が、あなたのゲートにある「時代区分（Era）」やデータトラックを自動検知できるようになります。

JSON
{
  "gate_code": "886",
  "country_name": "Taiwan",
  "maintainers": ["taiwan-digital-civics"],
  "eras": [
    {
      "era_id": "1996_digital_democracy",
      "era_name": "デジタル民主主義移行期",
      "policies_list": [
        "eras/1996_digital_democracy/policies/886_g0v_vtaiwan.json"
      ]
    }
  ]
}
② /eras/[era_id]/policies/[ファイル名].json（ファクト台帳）
これが、各国の政策やインフラの実績ログ（エビデンス）を格納する台帳の本体です。

JSON
{
  "policy_id": "886_vtaiwan_2015",
  "title": "vTaiwan（デジタル市民対話プラットフォーム）の法整備",
  "status": "implemented",
  "facts": {
    "execution_year": 2015,
    "primary_source_url": "https://vtaiwan.tw/about",
    "budget_allocated_usd": 0,
    "tags": ["Digital Democracy", "Civic Tech", "Consensus Building"]
  },
  "impact_logs": [
    {
      "metric": "Uber進出に伴う法改正の合意形成期間",
      "before": "不透明・長期化",
      "after": "4週間でステークホルダー間の合意形成を完了"
    }
  ]
}
🔗 3つの並列自律ロール（役割の選択）
KASHIKAに階層構造（上下関係）はありません。「特権管理者」も「一般ユーザー」も存在しません。参加者は自分の専門スキルに基づいて役割（ロール）を選択し、データログに直接、改ざん不可能な「実績と名誉」を刻み込みます。

      [ 🧠 ARCHITECT（接続特化） ] --------（ゲート間をポインタ結合し、矛盾を自動検知）
                   |
                   v
[ 📝 SCRIBE（記述特化） ] <=======> [ 🔍 AUDITOR（査読特化） ]
（1次ソースをJSONに写経）          （Facts Onlyの憲法を検証し、SBTを発行）
📝 SCRIBE（記述特化）
役割: 自国の公文書館、官報、歴史的記録からデータを抽出し、形容詞（主観）を徹底的に排除したクリーンなJSONファイル（上記のvTaiwanの例など）へと写経・翻訳する。

報酬（名誉）: 政策JSON内の "contributors" 配列に、あなたのGitHubアカウントが永久に刻印されます。その歴史的ファクトをデータ化した「最初の記述兵」として不滅の存在になります。

🔍 AUDITOR（査読特化）
役割: Scribe（記述兵）から提出されたプルリクエストを厳格に査読する。データに感情的な形容詞が混入していないか、1次ソースのURLが実在するかを検証し、マージを承認する。

報酬（名誉）: そのゲートの「真実の守護者」として、他人に譲渡・売買できないデジタルバッジ（SBT）を発行・署名する権限が与えられます。

🧠 ARCHITECT（接続特化）
役割: 異なるゲート同士の「格子」を緊密に結合する。例えば日本（kashika-gate-81）が新しい政策を計画した際、kashika-core 側にポインタを記述し、海外（kashika-gate-886 等）の過去データとクロス参照のパイプラインを繋ぐ。

報酬（名誉）: あなたが記述したマッピング定義が世界のデータグリッドの形そのものとなり、コアプロトコルの最上位コントリビューターとして世界的な名誉を獲得します。

⚡ 駆動原則：「過去の前例が、未来の政策を応援する」
データ構造が横並び（並列）で結合されることで、社会の意思決定は完全にリファクタリングされます。

政治論争の無効化: 「この政策は良い」「悪い」といった不毛な形容詞の殴り合いは不要になります。

構造的バックアップ（エビデンスの自動結合）: もし日本が2026年に「デジタル投票法案」を提出した場合、UI（画面）は自動的に台湾の2015年の vTaiwan ログや、エストニアの2005年の i-Voting ログをFetchして並列展開します。人間ではなく、データそのものがその政策の正当性を証明（あるいはバグを警告）します。
